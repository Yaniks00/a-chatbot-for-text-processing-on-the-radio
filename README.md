Running_Line — Telegram‑бот, который превращает обычные новостные тексты в готовые к эфиру радиосообщения. Он переписывает текст по правилам радио‑редактуры, расставляет ударения, рассчитывает хронометраж и подстраивается под ваш темп речи.

Возможности
Адаптация текста под эфир
Соблюдение жёстких правил: никаких причастных оборотов, сложных союзов или клише. Числа и даты — словами, расстановка ударений (выделение заглавной буквой).

Два стиля
Информационный — сухо, фактологично, строго.
Развлекательный — живая речь, лёгкие образы, разговорная интонация.

Контроль хронометража
Вы задаёте желаемую длительность (10–600 секунд) и темп речи (слов в минуту). Бот автоматически подгоняет объём текста, при необходимости сжимая или расширяя его.

Тайминг текста
Команда /timekeeping показывает, сколько времени займёт чтение вашего текста при выбранном темпе.

Расстановка ударений
Отдельная команда /stress выделяет ударные гласные заглавными буквами в словах и коротких фразах.

Настройки пользователя
Все параметры (стиль, темп, длительность) сохраняются в базе SQLite. Можно менять их в любой момент.

Юридическая прозрачность
Перед началом работы бот запрашивает согласие с пользовательским соглашением и политикой конфиденциальности. Есть команда /deletemydata для полного удаления всех данных.

Как это работает
Пользователь отправляет боту исходный новостной текст.

Бот формирует системный промпт для YandexGPT, учитывая выбранный стиль, темп речи и желаемую длительность. Промпт строго ограничивает объём ответа (диапазон слов, рассчитанный по формуле: темп × длительность / 60).

YandexGPT возвращает переписанный текст, соблюдая все редакторские правила.

Если объём ответа не попадает в заданный диапазон, бот автоматически делает повторный запрос на сокращение или расширение.

Готовый текст отправляется пользователю с техническими метками (стиль, темп, длительность).

Для расстановки ударений используется отдельный вызов модели с упрощённым промптом.
