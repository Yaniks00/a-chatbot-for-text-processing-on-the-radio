# Система адаптации новостных текстов для радиоэфира

Комплект инструментов для автоматической подготовки и адаптации новостных материалов под формат радиоэфира. Система состоит из двух независимых компонентов:

1. **Telegram-бот** (`bot.py`) – адаптирует тексты пользователей в реальном времени с учётом стиля, темпа речи и хронометража, используя YandexGPT.
2. **Анализатор аудио** (`analyzer.py`) – скачивает аудиовыпуски новостей из Telegram-канала, вырезает блоки по сигналам, распознаёт речь (Whisper) и генерирует статистический промпт для языковой модели на основе проанализированных текстов.

Оба компонента могут работать как вместе (промпт из анализатора используется ботом), так и по отдельности.


## Возможности
### Telegram-бот (bot.py)
- **Адаптация текста** под информационный или развлекательный стиль.
- **Контроль хронометража** – автоматический расчёт объёма текста под заданную длительность (10–600 секунд).
- **Настройка темпа речи** (50–300 слов/мин) с предустановками и точной регулировкой.
- **Расстановка ударений** в коротких фразах (выделение ударной гласной заглавной).
- **Расчёт хронометража** для любого текста без адаптации.
- **Сохранение пользовательских настроек** в SQLite.
- **Юридические документы** – ссылки на пользовательское соглашение и политику конфиденциальности.
- **Удаление данных** пользователя по запросу.

### Анализатор аудио (analyzer.py)
- **Скачивание аудио** из Telegram-канала (голосовые, аудиофайлы, документы с аудио-MIME).
- **Вырезание новостных блоков** по аудиосигналам начала и конца (корреляционный поиск).
- **Распознавание речи** с помощью OpenAI Whisper (модель `base`, язык русский).
- **Тематическая сегментация** расшифрованных текстов (алгоритм TextTiling на основе косинусной близости).
- **Извлечение признаков** для каждого блока:
  - количество слов, предложений;
  - длина предложений (средняя, мин., макс., стандартное отклонение);
  - индекс удобочитаемости Флеша (адаптированный для русского);
  - структура (вступление, основная часть, заключение) – количество слов и пропорции;
  - количество цифр;
  - (опционально) порядок слов (прямой/инверсия) – при установленной spaCy.
- **Агрегация статистики** по всем блокам и генерация **промпта** с рекомендациями для языковой модели (стиль, структура, длина, лексика, порядок слов).

## Технологии

- **Python 3.8+**
- **Telegram API** – `python-telegram-bot` (бот) и `Telethon` (анализатор)
- **YandexGPT API** – генерация адаптированных текстов (модель DeepSeek)
- **Whisper** – распознавание речи
- **pydub + scipy + numpy** – аудиообработка (вырезание, корреляция)
- **SQLite3** – хранение пользовательских настроек
- **spaCy** (опционально) – синтаксический анализ порядка слов
- **aiohttp, asyncio, tqdm** – асинхронность и прогресс-бары


## Установка и настройка

### 1. Клонируйте репозиторий
```bash
git clone https://github.com/yourusername/running-line.git
cd running-line
```

### 2. Создайте виртуальное окружение и установите зависимости
```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Содержимое `requirements.txt`:
```
aiohttp
yandexgpt-python
python-telegram-bot==20.7
nest_asyncio
python-dotenv
telethon
pydub
scipy
numpy
whisper
tqdm
# spacy  # опционально, для анализа порядка слов
```

Для работы с аудио может потребоваться **ffmpeg** (установите согласно вашей ОС).

### 3. Настройка переменных окружения (для бота)
Создайте файл `.env` в корне:
```
TELEGRAM_TOKEN=ваш_токен_бота
FOLDER_ID=ваш_folder_id_яндекс_облака
API_KEY=ваш_api_key_яндекс_облака
```

### 4. Подготовка юридических документов (для бота)
Загрузите на GitHub (или другой хостинг) файлы `user_agreement.md` и `privacy_policy.md`. Укажите их URL в коде `bot.py` (переменные `USER_AGREEMENT_URL` и `PRIVACY_POLICY_URL`).

### 5. Настройка анализатора
В файле `analyzer.py` укажите:
- `API_ID` и `API_HASH` – получите на [my.telegram.org](https://my.telegram.org).
- `CHANNEL_USERNAME` – ссылка или ник канала (например, `@news_channel`).
- `DOWNLOAD_LIMIT` – сколько последних сообщений сканировать.
- Пути к папкам (`DOWNLOAD_FOLDER`, `PROCESSED_FOLDER`, `TRANSCRIBED_FOLDER`).
- **Пути к сигналам** – создайте папку `signals/` и поместите туда свои файлы `start-signal.mp3` и `end-signal.mp3` (короткие джинглы, частота 22050 Гц).
- Параметры обработки: порог корреляции, окно поиска, макс/мин длительность, модель Whisper и т.д.


## Использование

### Запуск Telegram-бота
```bash
python bot.py
```

Команды бота:
| Команда           | Описание |
|-------------------|----------|
| `/start`          | Начало работы, принятие условий |
| `/parameters`     | Настройка стиля, темпа и хронометража |
| `/timekeeping`    | Расчёт времени звучания текста (можно указать темп аргументом) |
| `/stress`         | Расстановка ударений в короткой фразе |
| `/legal`          | Ссылки на юридические документы |
| `/deletemydata`   | Удаление всех пользовательских данных |
| `/cancel`         | Отмена текущего действия |

Просто отправьте боту любой текст – он адаптирует его согласно вашим настройкам.

### Запуск анализатора
```bash
python analyzer.py
```

Скрипт выполнит все этапы последовательно:
1. Скачает аудио из указанного канала.
2. Вырежет новостные блоки по сигналам.
3. Распознает речь.
4. Проанализирует тексты и создаст файлы `news_features.csv` и `prompt.txt` в корневой папке.

Полученный промпт можно использовать в качестве системного сообщения для YandexGPT (например, в боте). Для этого скопируйте содержимое `prompt.txt` и подставьте его в переменную `BASE_PROMPT` (или загружайте динамически).

## Как это работает (подробно)

### Бот
1. Пользователь отправляет текст.
2. Бот формирует системный промпт, учитывая выбранный стиль, темп речи и желаемый хронометраж.
3. Отправляет запрос к YandexGPT (модель DeepSeek) и получает адаптированный текст.
4. Проверяет количество слов в ответе; если оно не попадает в заданный диапазон, отправляет дополнительный запрос на сокращение или расширение.
5. Возвращает результат с пометкой о настройках.

### Анализатор
1. **Скачивание**: через Telethon перебирает последние сообщения канала, скачивает аудиофайлы в папку `downloaded_audio`.
2. **Вырезание**: для каждого файла загружает сигналы начала и конца, вычисляет корреляцию с аудиодорожкой, находит позиции, вырезает сегмент и сохраняет в `processed_audio`.
3. **Распознавание**: использует Whisper для транскрипции каждого вырезанного блока, сохраняет текст в `transcriptions` с той же иерархией.
4. **Анализ**: для каждого текста применяет сегментацию по темам (TextTiling), затем извлекает признаки (длина, сложность, структура, порядок слов). Все признаки агрегируются в CSV.
5. **Генерация промпта**: на основе сводной статистики создаётся текстовое описание идеального стиля, которое можно скопировать в системный промпт бота.


## Настройка сигналов для анализатора

Сигналы начала и конца должны быть короткими (1–3 секунды) и присутствовать в каждом аудиовыпуске. Для наилучшего распознавания:
- Используйте моно-файлы с частотой 22050 Гц (скрипт автоматически приводит к ней).
- Экспериментируйте с `CORRELATION_THRESHOLD` (по умолчанию 0.5), если сигнал плохо обнаруживается.
- Если сигнал конца отсутствует, скрипт ограничит блок параметром `MAX_NEWS_DURATION`.
